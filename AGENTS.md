# AGENTS.md

이 프로젝트에서 코딩 에이전트가 따라야 할 작업 지침입니다. `andrej-karpathy-skills`의 Karpathy-inspired Claude Code Guidelines를 참고해 이 타이머 프로젝트에 맞게 정리했습니다.

## 1. Think Before Coding

가정하지 말고, 헷갈리면 드러낸다.

- 구현 전에 요구사항과 성공 기준을 짧게 확인한다.
- 여러 해석이 가능한 요구사항은 조용히 하나를 고르지 말고 차이를 설명한다.
- 더 단순한 접근이 충분하면 그 방향을 우선한다.
- 모르는 상태로 코드를 크게 바꾸지 않는다.

## 2. Simplicity First

문제를 푸는 최소 코드가 우선이다.

- 이 프로젝트는 기본적으로 순수 HTML, CSS, JavaScript 단일 파일을 우선한다.
- 요청되지 않은 프레임워크, 번들러, 빌드 도구, 외부 음원 의존성을 추가하지 않는다.
- 한 번만 쓰는 코드를 위해 추상화를 만들지 않는다.
- 200줄로 쓴 기능이 50줄로 명확하게 가능하면 줄인다.

## 3. Surgical Changes

필요한 부분만 고친다.

- 사용자 요청과 직접 관련 없는 파일, 포맷, 문구를 바꾸지 않는다.
- 기존 스타일과 구조가 있으면 맞춘다.
- 내가 만든 변경으로 생긴 불필요한 코드만 정리한다.
- 관련 없는 죽은 코드나 개선 아이디어는 삭제하지 말고 필요하면 언급만 한다.

## 4. Goal-Driven Execution

명령을 검증 가능한 목표로 바꾼다.

- "타이머를 만든다"는 "브라우저에서 열었을 때 시작, 일시정지, 리셋, 반복 알람, 누적시간 초기화가 동작한다"로 검증한다.
- "옵션을 추가한다"는 "라이트/다크, 폰트, 반복 간격, 최대 시간, 알람 종류가 화면과 동작에 반영된다"로 검증한다.
- 변경 후에는 최소한 HTML 구조, 자바스크립트 문법, 주요 사용자 흐름을 확인한다.

## Project-Specific Success Criteria

- `index.html`은 파일을 직접 열어도 동작해야 한다.
- 기본 반복 간격은 1분 40초다.
- 반복 간격 변경은 현재 카운트다운을 새 간격으로 리셋하되, 총 누적시간은 유지한다.
- 최대 시간은 현재 세션의 총 누적 실행 시간 제한으로 처리한다.
- 알람 소리는 Web Audio API로 합성하며 외부 파일을 요구하지 않는다.
